Radiation¶

We need to specify the flux and thermal quantities used in the model, so let’s get right to it.

Incident Flux¶

We can define the beam flux and the isotropic flux at the top of the atmosphere in the radiation.IncidentFlux class. Instances of this class don’t do anything but containerize your desired flux, and provide some default values. I’ll go ahead and make an object and show its parameters.

n_layers = hydro.n_layers
scale_height = hydro.scale_height

from pyRT_DISORT.rayleigh import RayleighCO2

By default, beam_flux is pi, and isotropic_flux is 0. At least for our Martian simulation, there’s no real need to worry about the isotropic flux from space.

Thermal Emission¶

We can also define whether thermal emission is used in the model with the radiation.ThermalEmission class. As before, pyRT_DISORT has default values to get you up and running, though you can easily override them.

rayleigh_phase_function = short_rayleigh.phase_function
rayleigh_od = short_rayleigh.scattering_optical_depth

from pyRT_DISORT.controller import ComputationalParameters, ModelBehavior

By default, thermal_emission==False and no other values are used. But you’re free to turn thermal emission on and add values for the boundaries.
